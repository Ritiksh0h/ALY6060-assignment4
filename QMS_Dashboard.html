Act IV — model reliability across production lines
Why the same KPI behaves differently by production model
Aggregate results are strong. Split by model, relationships weaken — small production volumes inflate statistical uncertainty. This is the key limitation Industry 4.0 real-time data solves.
All models pooled Model A Model C Model D
The sample size trap: Model B has no confirmed regression relationship — not because quality relationships don't exist, but because low production volumes make coefficients statistically unreliable. Real-time IoT data pipelines eliminate this permanently by making n large enough to detect even weak signals.
What the data proves
One internal KPI — FTT — predicts customer warranty rate with 76.9% accuracy. A 2% improvement eliminates 0.4 warranty repairs per 1,000 units. The QMS is calibrated correctly; the problem is speed of feedback.
What Industry 4.0 changes
The 3-month lag compresses to hours. Sample size constraints disappear. Model B becomes analyzable. Cross-facility benchmarking becomes possible. The analytical logic stays identical — only the infrastructure accelerates it.
